How do mobile devices currently handle :hover?

  • If it's a link: right after you click, you will see the hover effect briefly before the page navigates away
  • If it's styled using either the CSS property of display or visibility: double-tap behaviour is needed

Workarounds

Current

  • Make all display:none or visibility:hidden hover states into a toggle situation using JS
  • Use media queries to determine when to disable or modify a default :hover effect based on screen size

But then what about all those sweet, sweet hovers???

Why not depend on media queries?

  • Touchscreen devices are getting bigger and bigger
  • Pre-emptively getting rid of a good hover effect affects user interaction
